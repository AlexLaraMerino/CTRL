# CTRL — Project Rules

## Product Rules
- Not an ERP
- Map is central
- Simplicity over features

## MVP Rules
Focus only on:
- map
- daily state
- notes
- employees
- work sites
- copy features

## Technical Rules
- Native iPad-first
- Swift everywhere in the iPad app
- Local first, backend later
- Use Apple frameworks when they improve UX quality

## UI Rules
- iPad-first
- Minimal UI
- Fast touch interactions
- Large map, lightweight overlays

## Domain Rules
- Each day is independent
- No cross-day side effects
- Copy is explicit

## Product Quality Rule
Interaction quality matters more than platform breadth in this phase.

## Branching
- Small branches
- One purpose per branch

## Success
Faster than a notebook, and fluid enough to trust in daily operation.
